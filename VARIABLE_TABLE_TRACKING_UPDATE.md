# PHP Analyzer Update Bundle / PHP Analyzer 개선 묶음

## 1) New additions in this round / 이번 라운드 추가 개선

### EN
Based on your latest feedback, the Risk Engine now provides three direct outputs:

1. **Query type analysis**
   - `db.query_locations[]` now includes `type` (`SELECT`, `INSERT`, `UPDATE`, `DELETE`).
   - Added `db.query_type_counts` with:
     - per-type counts (`SELECT`, `INSERT`, `UPDATE`, `DELETE`)
     - grouped counts (`read`, `write`)

2. **Table → query mapping**
   - Added `db.table_query_map` structure:
     - `{ "table_name": [{ line, type }, ...] }`
   - This enables direct impact navigation from a table to concrete query lines and query kinds.

3. **Automatic related file inference**
   - Added `db.related_files` generated by combining:
     - `dependency.inbound_paths`
     - file sets from `db.same_table_users`
   - This gives the AI/client a focused file list without broad repository search.

### KR
최신 피드백 기준으로 Risk Engine이 아래 3가지를 직접 제공합니다.

1. **쿼리 타입 분석**
   - `db.query_locations[]`에 `type`(`SELECT`, `INSERT`, `UPDATE`, `DELETE`) 추가.
   - `db.query_type_counts` 추가:
     - 타입별 건수(`SELECT`, `INSERT`, `UPDATE`, `DELETE`)
     - 그룹 건수(`read`, `write`)

2. **테이블 → 쿼리 매핑**
   - `db.table_query_map` 추가:
     - `{ "table_name": [{ line, type }, ...] }`
   - 테이블 영향 분석 시, 해당 테이블의 실제 쿼리 위치/종류를 바로 확인할 수 있습니다.

3. **related files 자동 추론**
   - `db.related_files` 추가.
   - 생성 기준:
     - `dependency.inbound_paths`
     - `db.same_table_users`의 파일 목록
   - 불필요한 전체 코드 탐색 없이 우선 확인 파일을 빠르게 좁힐 수 있습니다.

## 4) Known limitation / 현재 한계

### EN
- Query/table extraction is heuristic static analysis and may be partial for heavily dynamic SQL assembly.
- `related_files` prioritizes relevance but does not guarantee completeness for runtime-only paths.

### KR
- 쿼리/테이블 추출은 정적 분석 기반 휴리스틱이라 동적 조합이 복잡하면 일부 누락될 수 있습니다.
- `related_files`는 탐색 범위를 줄여주지만, 런타임 경로까지 100% 보장하진 않습니다.
